# Marketing Campaign ADK Agent

A `SequentialAgent` pipeline that turns a virtual-try-on image + a brand
guidelines PDF into a Google Ads Responsive Display creative set.

Pipeline: `brand_parser` → `creative_director` → `asset_generator`.

Output (per Google Responsive Display specs):

| Asset | Aspect | Pixel size | Generated by |
|---|---|---|---|
| Marketing image (landscape) | 1.91:1 | 1200 × 628  | Nano Banana edits VTO photo |
| Marketing image (square)    | 1:1    | 1200 × 1200 | Nano Banana edits VTO photo |
| Logo (landscape)            | 4:1    | 1200 × 300  | Imagen 4 |
| Logo (square)               | 1:1    | 1200 × 1200 | Imagen 4 |

## Layout

```
marketing_campaign_agent/
├── requirements.txt
└── marketing_campaign_agent/
    ├── __init__.py
    ├── agent.py             # SequentialAgent + 3 sub-agents + 2 tools
    └── .env.example
```

## Run standalone

```bash
pip install -r requirements.txt
cp marketing_campaign_agent/.env.example marketing_campaign_agent/.env
adk web                                     # from this directory
```

## Run as a sub-agent of the VTO agent

See [`../image_studio_root_agent/`](../image_studio_root_agent/) — that package mounts this
pipeline under the VTO root agent's `sub_agents` so a single session
flows: try-on → "make ads from this" → upload brand PDF → 4 assets.
